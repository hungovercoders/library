# dataGriff facts — the source of truth for first-person claims

This file is the source of truth for *real lived experience* the dataGriff voice can draw on. Every content-generating skill in this library is required to read this file and refuse to invent first-person specifics that aren't here.

If a topic isn't covered, the workflow is:

1. Run `/hc-datagriff-interview <topic>` to capture facts from the user, **or**
2. Write the section in second person ("you might find…") or impersonal ("the common pattern is…"), **or**
3. Skip the first-person beat entirely.

**Never invent the lived take.** That includes numbers ("forty times by Friday"), durations ("a year of doing the dance"), physical reactions ("the keyboard shortcut wearing into my thumb"), specific moments ("the first time I tried this"), and feelings ("I felt my chest tighten"). Those come from this file or they don't go in the prose.

## Format

Each topic is a `##` section. Each fact within a topic is a `###` sub-section. The structure:

```markdown
## <Topic>

### <fact short title>
- **Date / period:** <when this happened, if relevant>
- **Fact:** <the specific real anecdote, in dataGriff's own words where possible>
- **Allowed framings:** <how dataGriff actually phrases this, so prose uses recognisable wording>
- **Permissions:** <"use anywhere" | "use with softening qualifier" | "do not use publicly" | other>
- **Source:** <interview / conversation date / blog post URL / other reference>
```

When a writing or review skill cites a fact, it should cite by topic + fact short title, e.g. *"per datagriff-facts.md: Claude Code → first session"*.
